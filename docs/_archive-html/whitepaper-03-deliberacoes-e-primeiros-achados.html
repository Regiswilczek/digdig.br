Quando as Deliberações Falam Mais Alto

Regis Wilczek  —  Abril de 2026

Onde Paramos

No White Paper anterior documentei o incidente de custo: $23,72 consumidos por rodadas paralelas e workers zumbis, e as quatro camadas de proteção que implementamos para que não aconteça de novo — guard no endpoint, idempotência no serviço, verificação cooperativa de cancelamento no worker, e teto automático de $15 por rodada.

Com a infra corrigida, a rodada das portarias foi concluída. Os resultados finais:

400portarias analisadas
$8,87custo total rastreado
64%sinalizadas (amarelo+)

A distribuição das portarias foi: 35% verde (conformes), 64% amarelo (suspeitas), menos de 1% laranja (indício moderado). Nenhum vermelho.

O próximo passo natural era o segundo tipo de ato do CAU/PR: as deliberações. É onde as decisões plenárias ficam registradas — aprovações de orçamento, resultados de processos disciplinares, alterações de regimento, nomeações de comissões. Se as portarias são os atos de execução, as deliberações são os atos de poder.

O Primeiro Obstáculo: Dados Duplicados

O ponto de partida era um arquivo JSON gerado anteriormente com os metadados das deliberações coletados automaticamente do site do CAU/PR. O arquivo listava 1.480 itens.

Antes de prosseguir, rodei uma verificação de deduplicação:

import json
from pathlib import Path

with open("deliberacoes_completo.json") as f:
    dados = json.load(f)

numeros = [d.get("numero", "").strip() for d in dados]
unicos = set(n for n in numeros if n)

print(f"Total no JSON: {len(dados)}")
print(f"Números únicos: {len(unicos)}")
print(f"Duplicatas: {len(dados) - len(unicos)}")

Resultado:

Total no JSON: 1.480
Números únicos: 757
Duplicatas: 723

Quase metade do arquivo era ruído — a mesma deliberação raspada de diferentes seções do site, cadastrada múltiplas vezes com o mesmo número. O scraper original não deduplicava: se uma deliberação aparecia tanto na lista principal quanto em uma página de arquivo, ela entrava duas vezes.

O banco de dados tem uma constraint UNIQUE(tenant_id, numero, tipo). O script de seed ignorou silenciosamente os duplicados na inserção. O que chegou ao banco foram exatamente 757 deliberações únicas — que é o número real de atos distintos publicados pelo CAU/PR.

O Segundo Obstáculo: JavaScript

Com os metadados no banco, o próximo passo era descobrir as URLs dos PDFs para poder baixá-los e extrair o texto. O plano inicial era raspar a página de listagem de deliberações diretamente.

O problema: a página caupr.gov.br/?page_id=17916 é renderizada via JavaScript. Um GET direto retorna o HTML do WordPress com o conteúdo da página completamente ausente — o JavaScript que monta a lista de deliberações não executou. BeautifulSoup não vê nada.

A alternativa óbvia seria usar Playwright ou Selenium para renderizar a página com um browser headless. Mas isso adiciona dependências pesadas, demora, e traz seus próprios problemas de estabilidade.

Então olhei para a URL com atenção. Era uma página do WordPress. O WordPress tem uma API REST embutida.

GET https://www.caupr.gov.br/wp-json/wp/v2/pages/17916

A resposta JSON trouxe o campo content.rendered — o HTML completo da página, já com todos os links de PDF, sem precisar de JavaScript. Em uma única chamada HTTP.

1chamada à WP REST API
573URLs de PDF descobertas
0dependências extras

A estrutura do HTML retornado seguia um padrão consistente: cada deliberação era representada por uma sequência de parágrafos — um com o título em negrito contendo o número, seguido de até quatro parágrafos que podiam incluir data, descrição e o link para o PDF.

soup = BeautifulSoup(content_html, "lxml")
paragraphs = soup.find_all("p")

for i, p in enumerate(paragraphs):
    # Título: parágrafo com <strong> contendo número de deliberação
    if not p.find("strong"):
        continue
    texto = p.get_text(" ", strip=True)
    candidatos = normalizar_numero(texto)
    if not candidatos:
        continue

    # Procura link PDF nos próximos 4 parágrafos
    pdf_url = None
    for j in range(i, min(i + 4, len(paragraphs))):
        for a in paragraphs[j].find_all("a", href=True):
            if a["href"].lower().endswith(".pdf"):
                pdf_url = a["href"]
                break
        if pdf_url:
            break

    if pdf_url:
        await conn.execute(
            "UPDATE atos SET url_pdf=$1 WHERE id=$2",
            pdf_url, row["id"]
        )

Um detalhe não trivial: os números de deliberação no site às vezes aparecem sem zero à esquerda — "191-20/2025" em vez de "0191-20/2025" como estava cadastrado no banco. O código de correspondência testa primeiro o número exato, depois tenta com padding:

parts = numero.split("-", 1)
padded = parts[0].zfill(4) + "-" + parts[1]
# "191-20/2025" → "0191-20/2025"

Esse pequeno ajuste foi responsável por casar dezenas de deliberações que teriam sido perdidas na correspondência direta.

Fase 1: Download dos PDFs

Com 573 URLs descobertas, a próxima etapa foi baixar cada PDF e extrair o texto com pdfplumber. O mesmo scraper das portarias, adaptado para deliberações.

O servidor do CAU/PR bloqueia IPs de data centers americanos com 403 — o mesmo problema documentado no White Paper Nº 01. O scraper precisa rodar localmente, com IP brasileiro.

A extração passou por três execuções para processar o conjunto completo:

ExecuçãoProcessadasOK403/ErroTexto vazio
1ª5734188966
2ª (pendentes)8971126
3ª (pendentes)12921

Resultado final: 545 deliberações com texto extraído e pdf_baixado = true. As demais foram descartadas por PDF corrompido, resposta 403 persistente, ou PDFs escaneados sem camada de texto — o mesmo problema dos 151 portarias de 2018–2021.

Um detalhe prático: o Python em Windows usa cp1252 como encoding padrão no stdout. Os scripts usam caracteres de caixa UTF-8 nos logs de progresso, o que causava crashes silenciosos quando rodando em segundo plano. A correção é simples e vai no topo de cada script:

if sys.stdout.encoding and sys.stdout.encoding.lower() != "utf-8":
    sys.stdout.reconfigure(encoding="utf-8", errors="replace")

Montando a Rodada

Com 543 deliberações prontas para análise (545 com texto menos 2 com inconsistências), criamos a rodada de análise:

INSERT INTO rodadas_analise (
    id, tenant_id, status, total_atos,
    atos_analisados_haiku, custo_total_usd, criado_em
) VALUES (
    '5b365c0d-43be-4a7e-a22d-159f50fbe3c2',
    'f32ed0e7-c95d-4dec-a332-fa2cf6f20eb4',
    'em_progresso', 543,
    0, 0, NOW()
);

A rodada usa o mesmo motor de análise das portarias, mas roda localmente em vez do Railway — mesma razão do scraper: o Railway não tem acesso aos arquivos locais, e queríamos controle total sobre ritmo e custo durante essa fase exploratória.

A query de seleção é discriminada por tipo:

SELECT * FROM atos
WHERE tenant_id = $1
  AND tipo = 'deliberacao'
  AND pdf_baixado = true
  AND processado = false
ORDER BY data_publicacao ASC NULLS LAST

Ordenar por data crescente tem uma lógica: deliberações mais antigas tendem a estabelecer precedentes e padrões de gestão que contextualizem as mais recentes. O Dig Dig processa na ordem cronológica dos atos, não de inserção no banco.

Os Primeiros Números

Com a análise em andamento, os primeiros resultados foram chegando — e ficou imediatamente claro que deliberações são um tipo de ato fundamentalmente diferente das portarias.

Para comparação, a distribuição final das portarias:

E a distribuição dos primeiros 45 atos de deliberação analisados:

Nas portarias: 35% conformes, <1% laranja. Nas deliberações: 0% conformes, 47% laranja. O plenário deixa rastros mais visíveis do que os atos de execução.

A ausência de verde não significa que todas as deliberações são irregulares. Significa que o contexto das decisões plenárias — votações com quórum mínimo, ausências registradas, aprovações por ad referendum — cria, quase sempre, pelo menos uma observação que merece atenção. Verde exige que não haja nada a notar.

Três Padrões que Emergiram Cedo

1. O Abuso do Ad Referendum

Ad referendum é um mecanismo legítimo: quando o plenário não tem quórum ou quando há urgência, o presidente pode decidir sozinho e submeter a decisão à aprovação posterior do plenário. O regimento interno do CAU/PR prevê o uso.

O que o Dig Dig sinalizou repetidamente nas primeiras deliberações: o uso de ad referendum em decisões que não configuram urgência óbvia — aprovação de contratos, nomeação de comissões permanentes, remanejamento de verbas. Atos que poderiam esperar uma reunião plenária ordinária, mas foram decididos unilateralmente e homologados depois.

O padrão é sutil. Cada uso individual é defensável. Mas quando mapeados em série, a frequência sugere uma preferência sistemática por decisões executivas sobre deliberações coletivas — o oposto do que um conselho profissional deve ser.

2. Conflito de Interesse nas Sindicâncias

O segundo padrão envolveu deliberações sobre resultados de processos disciplinares. Em alguns casos, conselheiros que eram objeto de investigação — ou que tinham relação documentada com o investigado — participaram da votação que julgou o processo.

O regimento exige impedimento em casos de conflito de interesse. A ausência de registros de impedimento nas atas não significa que o conflito não existia — significa que não foi declarado.

Para o Dig Dig, isso é exatamente o tipo de irregularidade que um auditor humano encontraria consultando a composição do plenário na data do voto e cruzando com os nomes mencionados nos processos. O sistema não faz esse cruzamento de registros externos — mas sinaliza quando o texto da própria ata menciona nomes que aparecem simultaneamente como votantes e como partes interessadas.

3. A Deliberação 0102-11/2019

O único vermelho dos primeiros 45 atos analisados foi a deliberação 0102-11/2019, que recebeu score 9,0 numa escala de 0 a 10 e nível vermelho.

O Dig Dig identificou um conjunto de elementos que, isoladamente, seriam apenas suspeitos — mas combinados apontam para algo mais sério: uma sequência de deliberações que culminou na exoneração de um servidor com histórico documentado de denúncias internas, precedida por abertura de sindicância sem fundamentação explícita no texto, com votação realizada em reunião extraordinária convocada em prazo mínimo.

O padrão sugere perseguição institucional: uso do aparato administrativo — sindicância, reunião extraordinária, deliberação formal — para atingir uma pessoa específica. O Dig Dig não afirma isso — usa linguagem de "indício" e "padrão irregular". A conclusão jurídica pertence a um advogado.

Mas a flagração existe. E era exatamente isso que estávamos procurando.

Por Que Deliberações São Diferentes

Há uma diferença estrutural entre portarias e deliberações que explica a diferença nas taxas de sinalização.

Portarias são atos de execução: alguém nomeia, alguém exonera, alguém instala uma comissão. O texto é enxuto, padronizado, repetitivo. A irregularidade, quando existe, tende a ser direta — nepotismo evidente, cargo criado sem previsão, prazo violado.

Deliberações são atos de poder coletivo. O texto registra quem votou, quem se absteve, quem estava ausente, qual foi a motivação declarada para a decisão. Há argumentos e contrapontos. Há votações divididas. Há registros de quórum e de impedimentos — e a ausência de um registro de impedimento quando deveria existir é, ela mesma, uma informação.

É um texto muito mais rico para análise. E muito mais revelador.

Estado Atual e Próximos Passos

A triagem das 543 deliberações pelo Dig Dig está em andamento. O custo estimado para a rodada completa é de $6,52 — em linha com o custo das portarias, já que os textos têm tamanho similar.

543deliberações na rodada
~$6,52custo estimado (triagem)
$0,012custo por análise

Quando a triagem terminar, os casos vermelho e laranja passarão pela análise aprofundada do Dig Dig: ficha de denúncia estruturada, identificação de pessoas envolvidas. Com 47% de laranja nos primeiros 45 atos, essa fase pode ser significativamente mais extensa do que foi nas portarias — onde quase não houve casos críticos.

Os outros itens pendentes no pipeline:

  • OCR para portarias escaneadas — 151 portarias de 2018–2021 sem camada de texto precisam de Tesseract. Esse período cobre os anos anteriores às eleições do conselho e pode ser o mais relevante para padrões de longo prazo.
  • Deliberações HTML-only — algumas deliberações mais antigas existem apenas como HTML na página do CAU/PR, sem PDF. A Fase 3 do scraper lida com isso via BeautifulSoup diretamente na página de destino.
  • Dashboard conectado — o frontend em React já existe no ar. Falta ligar os endpoints da API para mostrar os resultados reais aos usuários.
  • Chat conversacional — RAG sobre os atos analisados, permitindo perguntas como "quais deliberações mencionam esse nome?" ou "mostre os casos de ad referendum entre 2020 e 2022".

A diferença mais importante entre portarias e deliberações não está nos números de sinalização. Está no que os números representam: portarias registram quem foi nomeado, deliberações registram por que. E o porquê — quando analisado em volume suficiente, com contexto suficiente — é onde as instituições se revelam.
